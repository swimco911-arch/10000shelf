Overview

프로그래밍 개요

Grand Shelf는 별도의 백엔드 서버 구축 없이 Google Sheets를 데이터베이스(CMS)로 활용하고, Firebase를 통해 사용자 인터랙션을 관리하는 웹 기반 인터랙티브 북셸프 애플리케이션의 아키텍처 문서입니다. 이 대시보드는 해당 아키텍처를 시각적으로 탐색할 수 있도록 구성되었습니다.

애플리케이션을 구성하는 핵심 기술 요소입니다. 각 카드를 클릭하여 상세 역할을 확인하세요.

2. 주요 아키텍처 및 데이터 흐름

  • 헤더 명칭 무관, 열 순서 기반 유연한 객체화 매핑 수행

2 인증 및 퍼머넌트 데이터

  • 익명 인증 지원 및 __initial_auth_token 커스텀 토큰 복구
  • Firestore 경로: /artifacts/{appId}/users/{userId}/interactions/{bookId}
  • 보안 규칙과 격리 준수

3. 상태 관리 로직 (State Management)

애플리케이션 내 핵심 데이터 배열들이 어떻게 결합되어 최종 UI를 구성하는지 보여주는 논리 구조입니다.

// 구글 시트에서 가져온 원본 도서 목록 배열
const [allBooks, setAllBooks] = useState([
  { id: 'book-1', title: 'React 입문', category: 'IT' },
  { id: 'book-2', title: 'Tailwind 완벽 가이드', category: 'Design' }
]);
// Firestore onSnapshot으로 동기화된 사용자별 상호작용(읽음 등) 상태 객체
const [userInteractions, setUserInteractions] = useState({
  'book-1': { isRead: true, updatedAt: '2026-03-13' },
  'book-2': { isRead: false, updatedAt: '2026-03-12' }
});
// useMemo를 사용해 상태 변경 시에만 도출되는 최적화된 결과 배열
const filteredBooks = useMemo(() => {
  let result = [...allBooks];
  
  // userInteractions 데이터를 매핑하여 필터링 적용
  if (filter.readStatus === '읽음') {
    result = result.filter(b => userInteractions[b.id]?.isRead);
  }
  
  return result;
}, [allBooks, userInteractions, filter]);

5. 성능 및 확장성 지표

애플리케이션의 렌더링 최적화(Memoization)와 반응형 환경에서의 그리드 제어 정책을 차트로 시각화합니다.

반응형 그리드 로직 뷰포트 너비에 따라 렌더링되는 책의 개수를 제한하여 모바일 환경의 DOM 오버헤드를 감소시킵니다 (Mobile: 5권 / Desktop: 10권).
렌더링 소요 시간 최적화 useMemo를 활용한 필터링 캐싱 처리로, 타이핑 검색 시 발생하는 무거운 재배열 연산 속도를 크게 단축했습니다.
